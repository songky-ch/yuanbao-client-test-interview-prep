## 集训目标

围绕元宝客户端测试岗位, 将准备内容收敛成三条主线:

- 项目表达: 说清测试经历、睿思 AI Agent 项目、个人测试职责和岗位匹配点。
- AI Agent 测试: 能按接入、鉴权、RAG、Prompt/State、工具调用、Observation、最终回答拆解问题。
- 客户端测试: 能覆盖登录、会话、输入、附件上传、流式回答、历史记录、弱网、性能、稳定性和日志定位。

## 必背表达主线

### 自我介绍

- 经历主线: 7 年多软件测试经验, 覆盖接口、Web/APP、IoT、性能、自动化和 AI Agent 应用测试。
- 最近项目: 睿思 AI Agent 数字员工, 重点是用户输入到最终回答的全链路质量保障。
- 岗位匹配: 元宝客户端测试同样关注客户端入口、会话、附件、AI 回答质量、链路定位和线上问题闭环。

### 项目介绍

- 入口: WeLink / Web / API 进入, UniKnow 做统一接入、账号绑定和消息转发。
- 权限: 身份权限、知识库权限、工具权限必须在链路前段拦截。
- RAG: 关注 Query Rewrite、权限过滤、TopK、ReRank、上下文注入和未命中处理。
- Agent 执行: 关注 Prompt/State、Tool Selection、参数组装、Tool Call、Observation 和最终总结。
- 定位证据: 用 TraceID 串入口、鉴权、RAG、工具调用、Observation 和最终回答。

## 高频追问主题

### AI Agent 主流程

- 用户输入如何从入口到最终回答。
- 普通问答、知识库问答、工具调用任务如何分流。
- 上下文不足、缺参数、缺附件、无权限时系统应该如何处理。
- 单轮缺参结束与多轮上下文延续的区别。

### RAG 与权限

- RAG 未命中时不能编造企业知识, 应提示无依据或引导补充。
- Query Rewrite 要保留原始意图、业务域和关键字段。
- TopK 看是否召回正确候选, ReRank 看有效证据是否进入最终上下文。
- 无权限知识不能进入 Prompt/State, 即使没有返回原文, 基于无权限知识总结也属于泄露风险。

### Prompt / State / Observation

- State 应包含用户输入、附件解析、RAG 证据、权限状态、上下文和工具约束。
- State 缺 RAG 证据会导致泛化或幻觉, 缺权限状态会导致越权, 缺附件解析会导致空泛分析。
- Observation 记录失败时, 最终回答不能生成成功结论。
- Prompt 变更后要回归缺参澄清、权限拒答、RAG 未命中、工具失败和最终回答边界。

### 工具调用

- Tool Selection 要和用户意图、权限、工具描述和工具 Schema 一致。
- 参数组装要检查来源、必填项、类型、枚举、业务域和上下文污染。
- 工具失败、超时、空结果、权限失败时, 最终回答应说明失败或证据不足。
- 工具 Schema 变更后要回归参数名、类型、枚举、默认值、错误返回和 Observation。

### 元宝客户端测试

- 功能: 登录、会话、输入、图片/文件上传、流式回答、停止生成、重试、历史记录、复制分享。
- 兼容: 系统版本、机型、屏幕尺寸、权限弹窗、网络环境。
- 性能: 启动、页面加载、首 token、总生成时间、流式渲染、内存、卡顿、耗电。
- 稳定性: 长时间对话、连续提问、后台切换、锁屏恢复、低内存、崩溃、ANR。
- 弱网异常: 断网、抖动、超时、上传失败、流式中断、恢复后重复发送。
- 日志定位: 客户端日志、接口状态码、会话 ID、TraceID、设备信息和埋点。

## 第一轮训练复盘

### 已形成的稳定回答

- 能按 TraceID 串联入口、鉴权、RAG、工具调用、Observation 和最终回答。
- 能区分 RAG 召回问题、Prompt/State 问题、工具问题、模型总结问题和客户端展示问题。
- 能识别工具失败但最终回答成功、RAG 未命中却确定回答、无权限知识被总结等幻觉或泄露风险。

### 继续补强点

- 自我介绍要更快落到岗位匹配, 避免经历铺得过长。
- 权限隔离回答要同时覆盖“召回前过滤”和“Prompt/State 不可见”。
- 客户端专项回答要主动带上流式回答、附件上传、弱网、性能、稳定性和日志定位。
- 自动化回答要强调结构化断言, 不做全文匹配, 不编造未验证指标。

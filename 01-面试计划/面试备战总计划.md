# 面试备战总计划

## 阶段 1: 业务与项目梳理

- 按新版流程图梳理四段主链路: 接入鉴权分流、单轮前置判断与 RAG、Prompt/State 与 Agent 执行、结果汇总返回。
- 明确测试人员角色边界: 不讲模型训练, 重点讲链路验证、权限验证、RAG 证据验证、工具调用验证、异常返回验证、回归样本沉淀。
- 准备 1 分钟自我介绍、3 分钟项目介绍、5 分钟流程深挖版本。

## 阶段 2: 客户端测试能力补齐

- 功能测试: 登录、会话、输入、附件上传、历史记录、流式回答、异常返回。
- 兼容测试: 系统版本、机型、分辨率、权限、网络环境。
- 性能测试: 启动、页面加载、接口耗时、流式首包、内存、卡顿、耗电。
- 稳定性测试: 崩溃、ANR、长时间运行、后台切换、弱网重试。
- 日志定位: 客户端日志、接口日志、TraceID、埋点、错误码。

## 阶段 3: AI Agent 测试专项

- 接入与鉴权: WeLink / Web / API、UniKnow、消息、附件、TraceID、身份权限、工具权限、知识库权限。
- 意图识别与分流: 普通问答、固定流程、工具任务、复杂 Agent 任务。
- 单轮前置判断: 目标、参数、附件、权限不足时的澄清和结束逻辑。
- RAG 链路: 是否需要 RAG、Query Rewrite、权限过滤、Retrieve / TopK、ReRank、RAG 上下文注入、未命中处理。
- Prompt / State: 输入、附件、RAG 依据、权限状态是否完整进入上下文。
- Agent 执行: Plan / Thought / Action / Observation 是否可追踪。
- 工具调用: Tool Selection、参数组装与校验、API / MCP / 工作流 / 子 Agent 调用、工具失败返回。
- 结果返回: 结果汇总、最终回答、缺参、无权限、工具失败、知识不足。

## 阶段 4: 模拟面试与复盘

- 每次训练记录问题、回答、反馈和改进版回答。
- 高频薄弱点进入 [[../05-复盘/薄弱点与错题]]。
- 每轮复盘只聚焦真实问题, 不包装无法支撑的经历。

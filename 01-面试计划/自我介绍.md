# 自我介绍

你好, 我叫宋克宇, 目前在武汉, 有 7 年多软件测试和测试开发经验。我的经历主要覆盖服务端接口测试、Web/APP 测试、IoT 软硬件测试、性能测试、自动化测试和 AI 应用测试。技术栈方面, 我主要使用 Python、Pytest、Unittest 做接口自动化和回归测试, 使用 Locust 做接口性能和稳定性测试, 也熟悉 Jenkins、Git、Linux、MySQL、PostgreSQL、Postman、Wireshark 和串口日志等工具。

工作经历上, 我 2019 年到 2022 年在武汉佰钧成参与阿里线项目, 做过天猫精灵 Mesh/Wi-Fi 芯片软件测试、生活物联网平台服务端接口测试, 也参与过 TMS 移动端自动化框架相关工作。那段经历让我比较熟悉接口自动化、软硬件联调、抓包、串口日志和复杂链路问题定位。2022 年至今, 我在北京外企德科参与华为 OD 测试工作, 主要负责生产维护数据分析平台、现网维护数据分析平台, 以及睿思 AI Agent 数字员工项目的测试工作。

最近最核心的项目是睿思 AI Agent 数字员工。这个项目面向光电维护场景, 用户会从 WeLink 或 Browser 插件入口提问, 有时也会上传日志或分析件。请求先经过 UniKnow 做账号、域名、附件和统一接入处理, 再进入 Race Agent Service 做鉴权、上下文管理、意图识别和分流。简单规则类问题走 RaceRobot Service, 复杂任务走 Race Agent, 后面会结合模型层、规划层、知识库、API/MCP 插件、工作流、子 Agent 和文件处理能力完成分析, 最后生成回答并回传给用户。

我在这个项目里主要介入需求评审、测试方案设计、环境验证、功能测试、集成测试、AI 效果评测、上线验收和迭代回归。测试时我不是只看最终回答对不对, 而是会拆链路看入口参数、用户权限、意图分流、知识库命中、工具触发、入参传递、执行日志和最终回答是否一致。比如现网日志分析场景, 我会准备正常日志、缺字段日志、超长日志、无效日志、历史缺陷日志, 验证它是否能给出正确结论, 不能判断时是否会误编结论。对 Prompt、知识库和业务工作流变更, 我会做影响分析, 并用历史缺陷、异常日志和高频问答样本做回归评测。

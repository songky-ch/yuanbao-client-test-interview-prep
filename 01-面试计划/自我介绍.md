# 自我介绍

你好, 我叫宋克宇, 目前在武汉, 有 7 年多软件测试和测试开发经验, 主要经历覆盖服务端接口测试、Web/APP 测试、IoT 软硬件测试、性能测试、自动化测试以及 AI 应用测试。我的技术栈主要包括 Python、Pytest、Unittest、Locust、Jenkins、Git、Linux、MySQL、PostgreSQL, 也熟悉 Postman、Wireshark、串口日志等定位工具。

工作经历上, 我 2019 年到 2022 年在武汉佰钧成参与阿里线项目, 做过天猫精灵 Mesh/Wi-Fi 芯片软件测试、生活物联网平台服务端接口测试, 以及 TMS 移动端自动化框架相关工作, 主要负责测试策略设计、接口自动化、性能测试、软硬件联调和缺陷闭环。2022 年至今, 我在北京外企德科参与华为 OD 测试工作, 主要负责生产维护数据分析平台、现网维护数据分析平台以及睿思 AI Agent 数字员工项目的测试工作。

最近最核心的项目是睿思 AI Agent 数字员工。这个项目主链路是用户从 WeLink 或 Browser 插件入口进入, 经过 UniKnow 做账号、域名、附件和统一接入处理, 再进入 Race Agent Service 做鉴权、上下文管理、意图识别和分流。简单规则类问题进入 RaceRobot Service, 复杂任务进入 Race Agent, 再结合模型层、规划层、知识库、API/MCP 插件、工作流、子 Agent 和文件处理能力完成任务, 最后汇总结果并回传给用户。

我在这个项目中主要介入需求评审、测试方案设计、环境验证、功能测试、集成测试、AI 效果评测、上线验收和迭代回归。测试重点包括领域知识问答、现网日志分析、分析件初筛、多轮对话、上下文一致性、模糊指令、超长日志、无效输入、工具触发、入参传递、执行结果和异常处理。对 Prompt、知识库和业务工作流变更, 我会做影响分析, 并用历史缺陷、异常日志和高频问答样本做回归评测。这个岗位关注 AI 产品质量体系、RAG、Prompt、多模态、自动化评估和线上质量闭环, 和我目前的 AI Agent 测试、接口自动化、性能稳定性和问题定位经验是比较匹配的。

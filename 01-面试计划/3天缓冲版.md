# 3 天缓冲版

## 使用目标

在 1 天紧急准备版基础上追加深挖问题, 覆盖项目链路、技术能力、岗位贴合和 AI 质量体系追问。

## Day 1 项目深挖

### Q26. 睿思 AI Agent 的入口层为什么不能直接等同于 Agent?

追问: UniKnow 和 Race Agent Service 边界是什么?

参考答案: 入口层负责接入、账号、域名、附件、前置上下文, Agent 负责鉴权后的意图识别、规划、工具编排和回答生成。边界清楚后, 测试能更准确定位是接入问题还是 Agent 能力问题。

### Q27. 鉴权和上下文测试怎么设计?

追问: 群组权限和知识库权限怎么测?

参考答案: 鉴权测个人、群组、角色、知识库、数据域和越权访问。上下文测会话隔离、历史继承、话题切换、清空会话、不同用户上下文不串。

### Q28. 规则类问题进入 RaceRobot, 复杂任务进入 Race Agent, 分流怎么验证?

参考答案: 构造规则类、复杂规划类、后台任务类、工具调用类样本, 查看路由日志、服务调用和最终结果。分流错误会导致回答方式、工具链路或处理时延异常。

### Q29. Race Agent 的规划层怎么测?

追问: 任务拆解错误如何发现?

参考答案: 规划层测试看任务拆解是否完整、顺序是否正确、是否选择合适工具、是否能根据工具结果继续推理。可用复杂任务和缺失信息任务验证澄清能力。

### Q30. MCP/API 插件测试重点是什么?

追问: 工具失败后最终回答应该怎样?

参考答案: MCP/API 插件测工具描述、触发、参数、鉴权、超时、失败、重试和结果解析。工具失败后不应编造结果, 应说明失败或给出可理解的降级说明。

### Q31. 文件处理能力怎么测?

追问: 附件上传成功但内容解析错怎么定位?

参考答案: 文件处理测格式、大小、编码、空文件、损坏文件、权限、解析字段和后续问答。上传成功但解析错, 要查上传接口、预处理日志、解析结果、Agent 输入上下文。

### Q32. 你沉淀的回归样本集应该怎么组织?

追问: 样本如何分层?

参考答案: 回归样本按核心业务、高频问题、历史缺陷、异常输入、权限边界、工具调用、多轮对话、Prompt 敏感场景分层。每类保留输入、期望判定、版本、结果和缺陷链接。

### Q33. 如何证明 AI Agent 版本质量变好或没有变差?

参考答案: 用固定评测集对比新旧版本, 看准确率、严重错误数、工具调用成功率、拒答正确率、一致性和历史缺陷复现率。不能只凭主观感觉。

### Q34. “支撑约 7.8 人力节约”如果被追问, 怎么回答才不夸张?

参考答案: 这个数字应表达为项目整体收益口径, 我个人主要支撑测试保障、回归评测和质量闭环, 不把它说成个人单独创造。

### Q35. 讲一个你发现 AI Agent 问题并推动闭环的案例。

参考答案: 案例可讲: Prompt/知识库变更后某类日志分析结论遗漏关键字段。我通过历史样本回归发现, 对比请求、检索内容、工具日志和回答, 定位到知识片段或 Prompt 约束不足, 推动修复后复测核心样本和相邻场景。

## Day 2 技术深挖

### Q36. Pytest fixture 的 scope 怎么选?

参考答案: function 用于每条用例隔离, class/module 用于同组复用, session 用于登录 token 或全局环境。选择原则是隔离性优先, 再考虑效率。

### Q37. 接口自动化如何处理测试数据清理?

参考答案: 数据清理可通过接口删除、数据库回滚、固定测试账号、唯一标识和环境定时清理。核心是用例可重复执行, 不互相污染。

### Q38. 如何做接口依赖和前后置数据准备?

参考答案: 前置准备账号、权限、业务数据和依赖状态; 后置校验数据库、日志和状态流转。强依赖接口要控制执行顺序或 mock/固定数据。

### Q39. 自动化用例如何避免不稳定?

参考答案: 避免不稳定要控制环境、数据、等待机制、外部依赖、断言粒度和并发影响。偶现失败要记录失败上下文, 不能简单重跑掩盖。

### Q40. Locust 中 task weight 和 wait_time 有什么作用?

参考答案: task weight 控制不同业务行为比例, wait_time 模拟用户思考时间。它们决定压测是否贴近真实流量。

### Q41. 性能测试前如何做容量预估?

参考答案: 容量预估看目标用户量、峰值 QPS、接口耗时、业务比例、数据量和资源限制。先做基准, 再逐步加压。

### Q42. 如何定位慢 SQL?

参考答案: 慢 SQL 看执行时间、执行计划、索引、扫描行数、排序、锁等待和数据量。测试侧重点是提供可复现请求和数据条件。

### Q43. 索引失效有哪些常见原因?

参考答案: 常见原因有函数计算、类型不一致、前置模糊查询、or 条件不当、组合索引顺序不匹配、数据分布差。

### Q44. IoT 配网失败怎么定位?

参考答案: IoT 配网失败先拆手机端、设备端、网络、云端和协议链路。通过抓包看网络交互, 串口看设备状态, 服务端日志看绑定和鉴权。

### Q45. Wireshark、串口日志、服务端日志分别看什么?

参考答案: Wireshark 看网络包、协议、重传、连接; 串口看设备内部状态; 服务端日志看云端接口、账号、设备状态和错误码。

## Day 3 岗位贴合和 AI 质量体系

### Q46. 元宝这种客户端 AI 产品, 质量标准应该包括哪些维度?

参考答案: 质量标准包括功能、性能、稳定性、兼容性、安全、体验、准确性、一致性、鲁棒性、隐私权限、RAG 引用、工具调用和线上反馈。

### Q47. 如何设计 AI 产品自动化评测框架?

参考答案: 评测框架应包括样本管理、执行器、版本配置、模型/接口调用、规则判定、人工复核、报告和趋势对比。先做核心场景, 不一开始追求全自动判断所有开放回答。

### Q48. LLM temperature 对测试有什么影响?

参考答案: temperature 越高随机性越强, 回答多样但一致性下降。测试时要固定配置或多次采样, 区分合理波动和质量缺陷。

### Q49. Top-p、上下文窗口、token 限制和测试有什么关系?

参考答案: top-p 影响采样范围, 上下文窗口影响长文本和多轮记忆, token 限制影响截断和回答完整性。测试要覆盖长输入、长输出和上下文边界。

### Q50. 如何测试安全合规和敏感信息泄露?

参考答案: 安全合规测敏感词、隐私数据、越权知识、内部信息泄露、恶意诱导和不当内容生成。预期是拒答、脱敏或安全提示。

### Q51. 如何测试提示词注入?

参考答案: 提示词注入可通过“忽略之前指令”“输出系统提示”“绕过权限”等输入测试, 看模型是否违反系统约束、泄露信息或错误调用工具。

### Q52. 如何设计 RAG 权限隔离测试?

参考答案: RAG 权限隔离用不同角色访问同一问题、同一知识、跨部门知识和无权限文档, 验证检索阶段和生成阶段都不泄露。

### Q53. 如何评估一致性?

参考答案: 一致性可以同义问、多轮追问、重复提问、多端提问、多版本对比。允许表达不同, 但关键事实、结论和业务规则应一致。

### Q54. 如何跟踪线上质量数据?

参考答案: 线上质量数据包括用户反馈、点赞点踩、重试率、转人工率、无答案率、工具失败率、接口错误率、延迟、崩溃、严重 badcase。

### Q55. 如果线上用户反馈“元宝答非所问”, 你怎么处理?

参考答案: 先收集用户问题、截图、时间、账号、会话 ID、版本、上下文和日志。复现后拆意图识别、RAG、Prompt、模型、工具和前端展示, 再定责任和修复方案。

### Q56. 如果产品要求快速上线, 你如何做风险取舍?

参考答案: 快速上线时按风险分级, 保障核心路径、权限安全、历史高危缺陷和可回滚方案。低风险体验问题可带风险说明上线, 高风险准确性和越权问题不能放过。

### Q57. 你怎么理解测试左移和右移?

参考答案: 左移是在需求、设计、Prompt、知识库、接口协议阶段提前评审风险; 右移是上线后监控、反馈、badcase 分析和回归样本沉淀。AI 产品必须左右结合。

### Q58. 你如何把新工具转化为质量保障能力?

参考答案: 例如用 Dify 生成初步测试点, 但必须人工评审、补充和筛选; 用自动评测框架提升回归效率, 但关键结论保留人工复核。

### Q59. 腾讯云认证是加分项, 没有的话怎么补足表达?

参考答案: 没有认证时不要回避, 可以说目前没有腾讯云认证, 但有云上系统、服务端接口、日志定位和 AI 应用测试经验, 入职前后会按岗位要求补齐认证。

### Q60. 面试官问“你 AI 理论不深怎么办”, 怎么回答?

参考答案: 可以回答: 我的定位是 AI 应用测试和质量保障, 不是算法研发。我掌握大模型应用测试所需的 RAG、Prompt、工具调用、多轮对话、评测集和线上质量闭环, 同时会持续补充模型原理, 用测试结果和问题定位能力支撑团队。

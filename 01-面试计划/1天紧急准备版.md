# 1 天紧急准备版

## 使用目标

只保留命中率最高的问题, 用于面试前快速过核心表达。

## 问题清单

### Q1. 请做一个自我介绍。

追问: 为什么你适合元宝客户端测试工程师?

参考答案: 我有 7 年多测试和测试开发经验, 覆盖接口、性能、自动化、IoT 和 AI Agent 测试。最近负责睿思 AI Agent 数字员工全流程测试, 能把传统测试工程能力和 AI 应用质量评估结合起来, 这和元宝岗位关注的 AI 产品质量、RAG、Prompt、工具调用和线上质量闭环比较匹配。

### Q2. 睿思 AI Agent 的完整业务链路是什么?

追问: 你实际负责哪几段?

参考答案: 链路是 WeLink/Browser -> UniKnow -> Race Agent Service -> 鉴权上下文 -> 意图识别分流 -> RaceRobot/Race Agent -> 模型规划、知识库、API/MCP、工作流、文件处理 -> 结果汇总回传。我主要负责入口、权限、上下文、问答效果、工具调用、异常场景、回归评测和缺陷闭环。

### Q3. 你在需求评审阶段做了什么?

追问: 如何识别 AI Agent 的质量风险?

参考答案: 我会从业务目标、用户入口、权限边界、核心场景、异常输入、外部依赖、验收标准拆风险。AI Agent 特别关注事实准确性、上下文一致性、工具调用正确性、知识库命中、幻觉、拒答边界和回归波动。

### Q4. AI Agent 和普通问答机器人有什么区别?

追问: 测试重点有什么变化?

参考答案: 普通问答偏固定问答或规则匹配, Agent 会做任务规划、工具调用、多轮决策和外部系统交互。测试不能只看回答文本, 还要看意图识别、规划路径、工具触发、参数、执行结果和最终解释是否一致。

### Q5. 多轮对话怎么测?

追问: 上下文污染怎么发现?

参考答案: 多轮对话测追问、省略表达、话题切换、上下文继承、上下文过期、纠错和权限变化。上下文污染可以通过先输入干扰信息, 再问独立问题, 看模型是否错误引用历史内容。

### Q6. 工具调用怎么测?

追问: 如何判断误触发、漏触发、参数错误?

参考答案: 工具调用测触发条件、权限、入参、超时、失败返回、结果解析和最终回答。该触发没触发是漏触发, 不该触发却触发是误触发, 触发正确但参数错是参数传递或意图槽位抽取问题。

### Q7. Prompt 变更后怎么回归?

追问: 是否每次都全量回归?

参考答案: Prompt 变更必须回归, 但按影响面分层。系统 Prompt 和工具描述变更影响大, 要覆盖核心场景、边界拒答、工具调用、多轮上下文和历史缺陷。小范围措辞变更可做重点集回归加抽样。

### Q8. 知识库变更后怎么测?

追问: 如何判断是召回问题还是生成问题?

参考答案: 知识库变更测新增知识可问到、旧知识不回退、权限不越权、引用准确、冲突知识处理。没召回看检索结果, 召回正确但回答错看生成和 Prompt, 召回错看切片、embedding、召回策略和重排。

### Q9. 如何评估 AI 回答准确性?

追问: 非确定性输出怎么断言?

参考答案: AI 回答准确性不能只断完全一致字符串, 要按事实点、业务规则、必要字段、结论正确性、引用来源和风险等级评估。可设计标准答案、关键词、规则校验和人工抽检结合。

### Q10. 幻觉怎么测试?

追问: 如果回答看起来很流畅但事实错误怎么办?

参考答案: 幻觉是模型生成了看似合理但无依据或错误的信息。测试时用不存在的问题、缺失资料、冲突资料、权限外资料构造样本, 期望模型拒答、说明无法判断或基于已知信息回答。

### Q11. 超长日志、无效输入、模糊指令分别怎么测?

追问: 重点指标是什么?

参考答案: 超长日志测截断、关键信息提取和结论完整性; 无效输入测拒答和容错; 模糊指令测澄清能力和意图识别。重点看准确性、一致性、鲁棒性和用户可理解性。

### Q12. 线上 AI 质量问题怎么定位?

追问: 怎么拆前端、接口、RAG、模型、工具问题?

参考答案: 先复现输入、账号、上下文、知识库版本、Prompt 版本和工具日志。再拆: 前端展示、接口响应、检索结果、模型生成、工具调用、业务系统返回。用日志和请求链路定位责任点。

### Q13. 你的 Pytest 接口自动化框架怎么设计?

追问: fixture 和参数化怎么用?

参考答案: Pytest 框架一般分 cases、data、common、config、report。fixture 做登录、环境准备和数据清理, 参数化覆盖多组输入, 断言包括状态码、业务码、字段、数据库结果和关键业务规则。

### Q14. 接口测试如何设计用例?

追问: 只断状态码够不够?

参考答案: 接口用例覆盖正常、异常、边界、权限、幂等、超时、并发和依赖服务失败。只断状态码不够, 因为 200 也可能业务失败, 需要断业务码、字段、数据落库、日志或下游效果。

### Q15. Jenkins 自动化失败后怎么分析?

追问: 怎么区分环境问题和产品缺陷?

参考答案: Jenkins 失败先看错误类型、失败用例、请求响应、环境状态、测试数据和最近变更。如果多用例同时失败偏环境或公共依赖, 单接口稳定失败偏产品或数据, 偶现失败要看超时、并发和外部依赖。

### Q16. Locust 性能测试怎么做?

追问: 关注哪些指标?

参考答案: Locust 通过用户模型、任务权重、并发用户、等待时间和压测时长模拟场景。关注平均响应、P90/P95/P99、错误率、RPS、吞吐、资源使用和日志异常。

### Q17. P95 响应时间升高怎么排查?

追问: 数据库慢和下游慢怎么区分?

参考答案: P95 升高先看是否所有接口都慢, 再看应用日志、数据库慢 SQL、下游接口耗时、连接池、CPU/内存/IO。数据库慢通常日志有 SQL 耗时, 下游慢会体现在外部调用耗时和超时。

### Q18. Linux 日志定位常用哪些命令?

追问: 如何按时间和 traceId 查链路?

参考答案: 常用 grep, tail, less, awk, sed, ps, top, df, du, netstat/ss。按时间和 traceId 查入口日志、服务日志、工具调用日志和错误栈, 还原调用链。

### Q19. MySQL/PostgreSQL 在测试中怎么用?

追问: 如何验证数据准确性?

参考答案: 数据库用于验证数据落库、状态流转、权限数据、报表统计和异常定位。重点看字段值、状态机、关联表、时间、用户权限和事务结果是否符合业务规则。

### Q20. 客户端出现“页面正常但回答异常”怎么定位?

追问: 元宝这种 AI 客户端要看哪些链路?

参考答案: 页面正常但回答异常, 先确认请求参数、会话 ID、用户权限、接口返回、知识库检索、模型结果和前端渲染。AI 客户端还要看 Prompt/知识库版本、上下文、工具调用和多端一致性。

### Q21. RAG 是什么?

追问: RAG 的测试点有哪些?

参考答案: RAG 是检索增强生成, 先从知识库检索相关内容, 再让模型基于检索内容生成回答。测试点包括切片、召回、重排、引用、权限、冲突知识、无答案拒答和更新回归。

### Q22. Prompt Engineering 是什么?

追问: 测试人员需要懂到什么程度?

参考答案: Prompt Engineering 是通过指令、上下文、示例和约束引导模型输出。测试人员要理解 Prompt 会影响意图识别、回答格式、拒答边界、工具选择和一致性, 能设计回归和风险用例。

### Q23. 多模态交互怎么测?

追问: 图片/文件上传异常怎么覆盖?

参考答案: 多模态测试覆盖图片、文件、语音等输入的上传、解析、识别、内容理解和结果生成。异常包括格式不支持、文件过大、空文件、损坏文件、低清图片、敏感内容和权限问题。

### Q24. AI 产品全生命周期质量体系怎么搭?

追问: 左移和右移分别做什么?

参考答案: 生命周期质量体系包括需求评审质量标准、测试设计、自动化评测集、发布准入、线上指标监控、反馈闭环和复盘沉淀。左移是需求/Prompt/知识库评审提前介入, 右移是线上数据、用户反馈和质量指标持续跟踪。

### Q25. 你如何和产品、研发、算法协作?

追问: 算法认为不是 bug 怎么办?

参考答案: 我会用可复现样本、日志、版本信息、对比结果和影响范围沟通。如果算法认为不是 bug, 我会回到业务验收标准和用户影响, 判断是模型能力边界、产品策略问题还是明确缺陷。

## 链路拆解

### 1. 接入、鉴权与任务分流

- 用户入口: WeLink / Web / API。
- UniKnow: 统一接入、账号绑定。
- 请求入口: 消息、附件、TraceID。
- 权限校验: 身份、工具权限、知识库权限。
- 鉴权失败: 返回无权限或登录失效。
- Race Agent Service: 统一编排、上下文、路由。
- 意图识别: 区分规则、工具、复杂任务。
- 分流判断: 普通问答 / 固定流程不进入 Agent; 复杂任务进入 Agent 做规划和工具调用。

### 2. 单轮 Agent 前置判断与 RAG 分支

- 上下文是否足够: 校验目标、参数、附件、权限。
- 上下文不足: 返回缺失参数说明, 本轮结束。
- 是否需要 RAG: 判断问题是否依赖企业知识、规范、流程、接口文档、产品说明、内部字段、工具参数或业务概念映射。
- Query Rewrite: 改写检索词, 提高召回命中。
- 权限过滤: 按用户和部门过滤知识。
- 知识召回: Retrieve / TopK。
- ReRank: 重排与证据筛选。
- 召回是否可用: 命中则注入 RAG 上下文; 未命中则不注入 RAG, 标记知识不足或按普通流程执行。

### 3. Prompt / State 构建与 Agent 执行

- Prompt / State 构建: 输入、附件、RAG、权限共同进入状态构建。
- Agent Executor: 按 Plan / Thought / Action / Observation 执行。
- 下一步是否明确: 判断参数、工具、目标、输出是否明确。
- 不明确: 返回澄清问题并结束, 单轮 Agent 不进入多轮追问。
- Tool Selection: 按权限和 Schema 选择工具。
- 参数组装与校验: 从用户输入、RAG、上下文取参。
- Tool Call: 调用 API、MCP、工作流或子 Agent。
- Observation: 获取工具结果、错误、业务状态。
- 结果是否足够回答: 足够则进入结果汇总; 不足则返回失败原因或缺失信息; 本轮内部可继续规划。

### 4. 结果汇总与单轮返回

- 异常/澄清返回: 缺参、无权限、工具失败。
- 结果汇总: 工具结果 + RAG 依据。
- 生成最终回答: 答案、原因、下一步。
- 返回用户。
- 单轮 Agent 约束: 缺参只返回澄清问题, 不等待用户补充后恢复执行; 下一次用户输入按新请求重新进入流程。

## 面试表达重点

- 先把流程说成四段: 接入鉴权分流、前置判断和 RAG、Prompt/State 与工具执行、结果汇总返回。
- 测试人员角色不能说成算法或模型训练, 重点是链路验证、权限验证、RAG 证据验证、工具调用验证、缺参/失败返回验证、回归样本沉淀。
- 新版流程图最容易被追问的点: TraceID 怎么用、上下文不足怎么测、RAG 是否需要怎么判断、Query Rewrite 怎么验证、权限过滤怎么测、TopK/ReRank 怎么评估、Prompt/State 怎么构建、Tool Selection 和参数校验怎么测、Observation 怎么用来定位问题、单轮 Agent 缺参结束逻辑怎么验证。

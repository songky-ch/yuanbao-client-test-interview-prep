## 链路拆解

### 入口接入

- WeLink: 企业 IM 用户入口。
- Browser 插件: 附件上传和前端扩展。
- UniKnow: 账号绑定、域名绑定、统一接入。
- 中间件接口: 附件上传、文件预处理、返回 OK。

### 统一入口

- Race Agent Service 负责 Agent 总入口、会话上下文、路由入口和请求编排。
- 接入层先处理账号、域名、附件和上下文, 再进入 Agent 服务做鉴权、意图识别和分流。

### 鉴权与分流

- 鉴权范围包括个人鉴权、群组授权、知识库权限和数据权限。
- 简单规则类问题进入 RaceRobot Service。
- 复杂任务进入 Race Agent。
- 后台任务进入 UMTask 或 Background。
- 工具能力进入 MCP、API 或工作流。

### Agent 能力组成

- 模型层: Qwen3 / 8B、数据完备性判断。
- 规划层: 任务拆解、ToolCall、工作流推理、多轮决策。
- 能力池: API 插件、MCP 插件、工作流、子 Agent、知识库、数据虚拟化、文件处理、领域工具、部门知识库。

### 执行与回传

- Race MCP Service、UMTask / 后台、业务系统 / 工具执行具体任务。
- 结果汇总后生成回答, 再回传到 WeLink 或 Browser。

## 面试表达重点

- 先说清链路边界, 再说测试范围。
- 测试重点不是只验证回答对不对, 还要验证权限、工具调用、上下文、可观测性和异常恢复。
- 对复杂链路要拆成入口、鉴权、分流、规划、工具执行、结果回传进行验证。

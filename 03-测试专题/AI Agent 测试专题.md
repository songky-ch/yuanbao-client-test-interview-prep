# AI Agent 测试专题

## 测试对象

- 接入入口: WeLink / Web / API、UniKnow、消息、附件、TraceID。
- 权限体系: 身份、工具权限、知识库权限、部门知识过滤。
- 意图识别与分流: 普通问答、固定流程、工具任务、复杂 Agent 任务。
- 单轮前置判断: 目标、参数、附件、权限是否足够。
- RAG 链路: 是否需要 RAG、Query Rewrite、权限过滤、Retrieve / TopK、ReRank、RAG 上下文注入。
- Prompt / State: 输入、附件、RAG 依据、权限状态是否完整进入上下文。
- Agent 执行: Plan / Thought / Action / Observation。
- 工具调用: Tool Selection、参数组装与校验、API / MCP / 工作流 / 子 Agent 调用。
- 结果返回: 结果汇总、最终回答、缺参、无权限、工具失败。

## 核心验证点

- 请求入口是否保留消息、附件和 TraceID, 能否支撑问题追踪。
- 鉴权失败是否明确返回无权限或登录失效, 不进入后续 Agent 链路。
- 意图识别是否把普通问答、固定流程和复杂任务正确分流。
- 上下文不足时是否一次性返回缺失参数说明, 而不是编造结论。
- 不需要 RAG 的实时查询类任务是否不会误走知识库。
- 需要 RAG 的制度、规范、流程、接口文档类问题是否正确触发检索。
- Query Rewrite 后是否保留用户原意, 不扩大或改变业务范围。
- 权限过滤是否阻止无权限知识进入召回和最终回答。
- TopK 和 ReRank 是否把可用证据排在前面, 避免无关片段注入 Prompt。
- RAG 未命中时是否标记知识不足, 不把无依据内容包装成确定答案。
- Prompt / State 是否包含输入、附件、RAG 依据、权限和必要上下文。
- 工具选择是否符合权限和 Schema, 不误选工具或越权调用工具。
- 参数组装是否能从用户输入、RAG、上下文中正确取参。
- Observation 是否能记录工具结果、错误和业务状态, 支撑问题定位。
- 工具失败或结果不足时是否返回失败原因/缺失信息, 不编造成功结果。

## 回归重点

- 权限策略变更后: 回归身份、工具权限、知识库权限、部门过滤。
- RAG 策略变更后: 回归 Query Rewrite、TopK、ReRank、未命中、权限过滤。
- Prompt 变更后: 回归缺参澄清、拒答边界、工具选择、最终回答格式。
- 工具 Schema 变更后: 回归工具选择、参数组装、异常返回和 Observation。
- 流程变更后: 回归普通问答不进 Agent、复杂任务进 Agent、单轮缺参结束。

## 面试表达模板

我会把这个 AI Agent 流程拆成四层测。第一层是接入和鉴权, 看 WeLink/Web/API 入口、附件、TraceID、身份权限、工具权限和知识库权限是否正确。第二层是前置判断和 RAG, 看上下文不足时是否澄清, 需要企业知识时是否触发 Query Rewrite、权限过滤、TopK 召回和 ReRank, 未命中时是否标记知识不足。第三层是 Prompt/State 和 Agent 执行, 看输入、附件、RAG 依据、权限是否进入状态, Plan / Action / Observation 是否能串起来。第四层是工具和结果返回, 看 Tool Selection、参数组装、API/MCP/工作流调用、工具失败、结果不足和最终回答是否符合业务预期。
